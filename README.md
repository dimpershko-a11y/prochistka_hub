# PRO-CHISTKA Hub

Единый модульный хаб для управления основными процессами клининговой компании PRO-CHISTKA.

## Назначение

Репозиторий создан как отдельная база для будущего объединения:

- калькулятора стоимости уборки;
- бизнес-дашборда;
- чек-листов;
- табеля сотрудников;
- ведомости повреждений;
- PDF-документов;
- CRM;
- заказов;
- расходов;
- сотрудников;
- партнёров;
- настроек.

## Главное правило архитектуры

Каждый раздел Hub должен быть отдельным модулем. Изменения внутри одного модуля не должны ломать остальные разделы.

Общие функции выносятся в `src/shared`:

- API;
- форматирование;
- расчёты;
- Google Sheets;
- PDF;
- настройки;
- хранение данных;
- общие UI-утилиты.

## Структура проекта

```txt
src/
  app/                 # ядро приложения, оболочка, роутинг, реестр модулей
  modules/             # независимые бизнес-модули
  shared/              # общие функции и инфраструктура
  assets/              # изображения, логотипы, иконки
  config/              # конфигурация Hub

docs/                  # архитектура, правила разработки, дорожная карта
```

## Модули

Стартовые модули:

- `calculator` — калькулятор стоимости уборки;
- `dashboard` — бизнес-дашборд;
- `checklists` — чек-листы работ;
- `timesheet` — табель сотрудников;
- `damages` — ведомость повреждений;
- `documents` — PDF-документы и шаблоны;
- `crm` — клиенты и коммуникации;
- `orders` — заказы;
- `expenses` — расходы;
- `employees` — сотрудники;
- `partners` — партнёры;
- `settings` — настройки Hub.

## Правила разработки

1. Новый функционал сначала добавляется в отдельный модуль.
2. Общий код нельзя дублировать внутри модулей — его нужно выносить в `shared`.
3. Модуль не должен напрямую менять данные другого модуля.
4. Связь между модулями должна идти через публичные методы, события или общий слой данных.
5. PDF, Google Sheets, форматирование и настройки не должны находиться внутри конкретного бизнес-модуля, если они используются повторно.

## Текущий статус

Репозиторий подготовлен как архитектурный каркас. Старый чат остаётся только для бизнес-дашборда, этот репозиторий и этот чат — только для PRO-CHISTKA Hub.

## Публикация в браузере

Приложение публикуется через GitHub Pages из production-сборки Vite.

После push в ветку `hub-v1` или `main` workflow `Deploy GitHub Pages` соберёт `dist` и опубликует сайт.

URL сайта:

```txt
https://dimpershko-a11y.github.io/prochistka_hub/
```

В настройках репозитория GitHub нужно выбрать `Settings → Pages → Source → GitHub Actions`.
